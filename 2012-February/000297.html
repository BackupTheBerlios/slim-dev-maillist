Summary: problem with focus_password

Details: In the config file, I write my login after default_user.

When I put the focus_password parameter on "yes", Slim dosen't log me. It seams like I have written a wrong user/password => SLIM ask me again my login and my password.

But if I enable only the parameter default_user (focus_password on "no"),  I obviously have to hit ENTER before writing my password but SLIM start my session correctly.

Comment:
The issue may be specific to the PAM inplementation, but the cause of the issue is that, when focus_password = yes, the keyboard event handler is not run on the username and at the present time the only place that "action" is set is within that event handler.

Patch 3207 sets action = Login when the username is set via the SetName function; after applying this patch it seems to work as expected.
