Summary: The keyboard doesn't work on the first console

Details: When I log into an xorg session - doesn't matter if it's KDE or Xfce - the keyboard seems to work in X, but if I switch to the console by pressing Ctrl-Alt-F1, the keyboard doesn't work anymore. I have to press very often onto one key, before the keyboard and bash react. It's so extreme that I can't do anything on the console, when I'm logged in an X session. And the German umlauts are only displayed as squares.

When I switch back to X by pressing Alt-F7, the keyboard is responding much better but not as fast as it did after the first X login.

As soon as I log out and only see slim the keyboard works correctly and reacts as fast as usual in slim as well as on the console.

I don't get any error messages.

This happens since Xorg server 1.4.2 or at least 1.5.2. And it happens only on the first console (vc/1 or tty1). On any other console the keyboard reacts as usual and expected.

Currently I'm using xorg-server 1.6.1.901, xf86-input-evdev 2.2.2 and slim 1.3.1.

See also these bug reports:
http://bugs.archlinux.org/task/12179
http://bugs.gentoo.org/show_bug.cgi?id=247543
http://bugs.freedesktop.org/show_bug.cgi?id=18622

In the meantime I don't think anymore, that this is a bug in the DEs or in Xorg but in SLiM, because I found out, that I don't get this problem, when using gdm.
