So xdm, kdm and gdm all support this. However, that argument style
doesn't work with getopt, so I can only see two possibilities:

1. switch to another way of parsing
2. add a hack like this:
    // accept nodaemon as '-n' argument to be compatible with xdm, gdm
    // and kdm
    for (i = 1; i < argc; ++i) {
        if (strcmp(argv[i], "-nodaemon") == 0) {
            *(argv[i]+2) = '\0';
        }
    }

Since we have a fairly small number of options, solution #1 would be my
preference. Opinions?
