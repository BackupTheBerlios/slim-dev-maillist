Hi,

> I finally completed some basic test of Martin implementation
> on netbsd. As previously mentioned, the openpam implementation
> lacks PAM_BAD_ITEM, PAM_FAIL_DELAY and pam_fail_delay().
> 
> The good news is there is no news: these are the only differences
> I found. Removing those bits produces a working, pam-linked slim
> binary.

Nice.

> Martin mentioned we can skip pam_fail_delay, while
> ifdef'ing PAM_BAD_ITEM should solve the other issue both on
> netbsd and freebsd. 

Actually I think it's a bad idea to ifdef on PAM_BAD_ITEM as this
(a) might change unexpectedly in future versions
(b) isn't clean to maintainers, why you exclude or include code

Instead I've been looking at Linux-PAM and OpenPAM and it seems they
both have defines we can use. Linux-PAM defines __LIBPAM_VERSION and
OpenPAM defines _OPENPAM, _OPENPAM_VERSION and _OPENPAM_RELEASE.

I suggest we use these defines instead, though I don't really care about
if we #ifdef __LIBPAM_VERSION or #ifndef _OPENPAM.
