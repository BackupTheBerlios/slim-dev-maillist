Summary: feature request: alternate pam identification methods

Details: Hi, I'd like to request a feature. It would be awesome if you could support alternative pam login methods. Some business model computers now have fingerprint readers and USB fingerprinter scanners and smart cards are popular enough. There's good support for these devices in linux and a very active community.

No other login manager handles alternative login methods very well, to my knowledge. XDM just makes you put in a blank password and then waits (for pam to respond) without saying anything. With the software I'm using, GDM pops up an ugly dialog box for scanning confirmation.

Making it look pretty would be easy, I imagine (though I have no clue). Something like two variables in the config file like "allow_altLogin" and "altLogin_text"... so if you set allow_altLogin it would display the altLogin_text, say like "Scan fingerprint..." instead of the password dialog box.
