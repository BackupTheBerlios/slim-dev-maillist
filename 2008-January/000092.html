On Wed, Jan 09, 2008 at 07:32:48AM +0100, Dennis Heuer wrote:
> i stumbled upon the slim.log file and mine was 11Mb big. please delete
> this log frequently.

I've noticed this as well. The culprit is that this log file picks up
the stdout/stderr of whatever has been launched, window manager, and the
not-grounded-in-xterm applications the window manager launches. This
accumulates quickly, and may imply that you're leaking private
information. You might want to redirect your session's output to
/dev/null in your xinitrc to counter this.
