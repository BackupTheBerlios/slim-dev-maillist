Summary: reboot and halt should optionally be possible without passwd

Details: reboot and halt is currently only possible by entering the root password. This is of course principally good for security reasons.
But it should also be possible to halt and reboot without entering a password.

This should of course be made optional. The option, whether a password is required or not, should be set in /etc/slim.conf.
