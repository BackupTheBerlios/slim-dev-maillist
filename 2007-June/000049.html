After doing some more testing myself I discovered that slim going
bananas if the PAM configuration is missing. The problem is that PAMs
default policy is to fail any authentication attempt if the requested
service is unconfigured however the applications using PAM is not given
any reason for the failure. So if the slim service is unconfigured slim
ends up in a rather nasty (and loud, if you have PC-speaker) loop.
So we DO need to include a configuration file.

It should be noted that this is a problem in PAM and NOT in slim. I have
a few ideas for fixes but I'll rather wait until after the 1.3.0 release
as I need to double check a few things with OpenPAM as well. I don't
consider it a show-stopper for the 1.3.0 release but it is something we
should be aware of if someone complaints about this behaviour.
