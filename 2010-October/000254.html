Summary: default session inconsistencies

Details: SLiM specifies that in its config file, if 'sessions' is present then by default the first in the list will be specified, but unless [F1] is pressed, the session is empty ("").  Also, it is not possible to select an empty session after [F1] has been pressed.

SLiM's behaviour should be consistent; either a value from this list should always be specified, or [F1] should allow selection of the default/empty session; and documentation in slim.conf should reflect this.
