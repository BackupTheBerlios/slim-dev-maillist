Dear SLiM developers,

For already quite some time I was looking for a nice and
dependency-less display manager. A couple of months ago, SLiM was
mentioned on the Xfce ML and I have started to use it. Since, I have
gathered a list of changes that would be worth implementing.

In this email I will address the first one: the halt/reboot commands.
The man page states: "The 'halt' and 'reboot' commands need the root
password, this may change in future releases."

I firmly believe that - unless it represents a serious risk hazard -
it is worth implementing this change. Ignoring that this is a tedious
way to halt / reboot your machine from within SLiM, IMO this feature
makes it difficult - next to impossible - to use SLiM in certain
environments.

I will cite the problem that I've come across: I installed Debian
Linux in an office environment for veritable end-users. It was
difficult to simply explain them the need to type halt / reboot in
order to accomplish those actions. It was however next to impossible
to explain them the need for a password in order to do this.


Regards,
Liviu
