Summary: numlock option broken in 1.3.3

Details: Enabling or disabling num lock with the "numlock" setting has been broken with the refactoring of numlock.cpp that happened in version 1.3.3. The cause is two-fold :
- code paths for "on" and "off" have been swapped when setOn and setOff have been coalesced into control_numlock.
- since a new X display object is no longer created when those functions are called, it is also no longer closed by XCloseDisplay(), which means that the implied XSync() call no longer happens.
