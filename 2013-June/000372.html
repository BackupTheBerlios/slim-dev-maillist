> Looking at your log:
>
> [    25.950] (**) FBDEV(3): claimed PCI slot 0 at 0:2:0
> [    25.950] (II) FBDEV(3): using default device
> [    25.950] (EE) Screen 0 deleted because of no matching config section.
> [    25.950] (II) UnloadModule: "vesa"
> [    25.950] (EE) Screen 0 deleted because of no matching config section.
> [    25.950] (II) UnloadModule: "modesetting"
> [    25.950]
> Fatal server error:
> [    25.950] Cannot run in framebuffer mode. Please specify busIDs
>    for all framebuffer devices'
>
> Basically you have an issue with the framebuffer driver... this driver
> is usually a fallback driver actually. Couple of stuff that we might
> need:
>
> * Could you post the configuration file from /etc/X11/xorg.conf.d? I
> am wondering if your device section is properly configured
> * lspci
> * What is the device you are planning to use for graphics
>
> Arch Wiki suggests to uninstall the fb driver if something like that
> happens... that might be worth the shot
> (https://wiki.archlinux.org/index.php/Xorg#Frame-buffer_mode_problems).
>
> Regardless this does not seem like a slim issue.
>
> Cheers,
