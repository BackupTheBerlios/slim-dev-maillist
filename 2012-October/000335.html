Summary: slim-1.3.4 (earlier?) -- pam session dies in child proc

Details: There is a call to pam.end() shortly after forking the child process, in app.cpp.  This is erroneous as it kills the active pam session.  In a default PAM setup this has no effect, but when PAM has been set up with kerberos integration, the kerberos ticket cache is cleaned.  This may also have an effect on SElinux systems.

This bug is the underlying reason why erroneous patches 2690 and (part of) 3043 have been submitted to attempt to fix the kerberos issue, however the actual fix is to use patch 3343
