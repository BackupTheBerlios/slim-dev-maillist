Howdy-

I recently started using xcompmgr in order to have translucent xterms.  Unfortunately, out of the box xcompmgr replaces the wallpaper set by SLiM with a solid gray background.

A little Googling turned up lots of suggestions to run hsetroot after xcompmgr to reset the wallpaper.  However, reading xcompmgr's source it turns out it will reuse an existing pixmap pointed to by an _XROOTPMAP_ID property.

I created a patch for SLiM that sets that property after setting the root window's background.  I uploaded it to the patches section of the BerliOS developer site, but it doesn't appear that automatically triggers an email to this list (hence this email).

The patch and a more detailed analysis can be found at http://developer.berlios.de/patch/?func=detailpatch&patch_id=3353&group_id=2663

Best,
Kent
