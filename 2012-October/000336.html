Summary: slim-1.3.4 - compilation fails on ARM, std::snprintf missing

Details: For whatever reason, ARM does not have std::snprintf available.  This is used only in the new Ck.cpp file, to ensure "static char device[32]" is not written past its bounds.

std::sprintf does however exist.  The operation to write to 'device' uses the format string '/dev/tty%ld' , and until 128-bit computing comes along this string will never be longer than 28 characters (+ null) and so will always be in-bounds.  I propose that the call to std::snprintf be changed to std::sprintf to support compilation on ARM.
