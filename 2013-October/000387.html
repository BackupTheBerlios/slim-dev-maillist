By: istakenv

Comment:
I've patched in the original session support in Gentoo's slim package, and have attached the patch I used here as patch #003513 ( http://developer.berlios.de/patch/index.php?func=detailpatch&patch_id=3513&group_id=2663 )

I've got a lot of other packages on the slim-1.3.6 codebase so I don't know if this one will apply clean out-of-the-box, but it should be close.

This patch restores support for the old 'session' config file option, as well as the old sessiondir behaviour.   It KEEPS the new sessiondir behaviour if the content of the sessiondir contains *.desktop files (that is, it contains files with Name= and Exec=)

It also removes the automatic session rotation on startup, that is, %session can be blank again if the end-user doesn't hit [F1] to cycle it.
