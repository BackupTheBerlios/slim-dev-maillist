Summary: Slim switches back to console before showing panel again

Details: Steps to reproduce:

1.) Log in using slim
2.) Log out of your X session

Notice how after logging out, the console is shown for a brief moment before slim's panel is shown again.

This isn't a serious issue, but it's kind of ugly. For example, xdm doesn't show this behaviour: when you log out of xdm, you get back to the xdm prompt immediately without console detours.

The problem might be that slim kills off all X clients when the user logs out (app.cpp:628,629) which shuts down the X server?
