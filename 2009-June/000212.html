repository of SLiM if you plan to submit your contributions; I've

> OK then, I could compile the code. Now my problems begin: How do I test
> the executable? Since I have slim already running in the background, it
> locks another instance. I tried just to comment out the lock stuff, but
> it would exit my window manager and just drop me to my old slim login
> like after a normal restart. I also had the feeling, my new executable
> is using my own slim theme even though I changed the path to the
> slim.conf.
> This is probably a noob question, but could you folks just tell me how
> you're testing slim?

It depends a lot on what you want to work on. If that's xserver
handling or early startup, you would typically not start slim in the
first place; I used two machine for convenience, but that's really a
matter of taste, i.e. how you like to develop.

If you want to work on UI stuff, there's the '-p' switch (preview
theme) which you can use; there's also a macro called XNEST_DEBUG
which you can #define to avoid the xserver handling as such.

Hope this helps,
